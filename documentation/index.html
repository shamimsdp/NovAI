NovAI Documentation

Everything you need to install, configure, and customise the NovAI WordPress block theme for AI startups, SaaS products, and developer-first companies.

v1.0.0 WordPress 6.4+ PHP 8.0+ FSE

Requirements

Software Minimum Version Recommended
WordPress 6.4 6.6+
PHP 8.0 8.2+
Browser Any modern browser Chrome, Firefox, Safari, Edge
Required Plugins None. WP All Import for demo content (optional).
Note: NovAI requires the WordPress block editor (Gutenberg) and Full Site Editing (FSE) support. It is not compatible with classic PHP themes or page builders.

Installation

Method 1 — WordPress Admin (Recommended)

  1. Log in to your WordPress admin dashboard.
  2. Go to Appearance → Themes → Add New.
  3. Click Upload Theme and select the novai.zip file.
  4. Click Install Now, then Activate.

Method 2 — WP-CLI

wp theme install novai.zip --activate

Method 3 — Manual Upload

  1. Extract novai.zip.
  2. Upload the novai/ folder to /wp-content/themes/ via FTP/SFTP.
  3. In WordPress admin, go to Appearance → Themes and activate NovAI.

Demo Content Import

NovAI ships with a demo content XML file covering all three starter templates.

  1. Install and activate the WP All Import plugin.
  2. Go to All Import → New Import.
  3. Upload demo/content.xml from the theme package.
  4. Follow the import wizard. Map fields as prompted.
  5. After import, assign the front page in Settings → Reading → Your homepage displays → A static page.
Important: Import demo content on a fresh WordPress installation to avoid conflicts with existing content.

Typography

NovAI uses a self-hosted Inter variable font — no Google Fonts, no CDN, GDPR-compliant.

  • Inter Regular 400 — body text, UI labels
  • Inter SemiBold 600 — headings, CTAs, badges

The font scale uses CSS clamp() for fluid sizing — text scales smoothly between mobile and desktop without media query breakpoints.

Using the Site Editor

  1. Go to Appearance → Editor.
  2. Use the Templates panel to edit page templates.
  3. Use the Patterns panel to insert pre-built sections.
  4. Use Styles → Browse Styles to switch between the 5 color schemes.
  5. Use Styles → Edit Styles to customise Global Styles (colors, fonts, spacing) without writing CSS.

Creating a Child Theme

To safely customise NovAI without losing changes on update, create a child theme:

Create a folder novai-child/ in /wp-content/themes/ with this style.css:

/*
Theme Name: NovAI Child
Template:   novai
Version:    1.0.0
*/

And a minimal functions.php:

<?php
add_action( 'wp_enqueue_scripts', function () {
    wp_enqueue_style(
        'novai-child-style',
        get_stylesheet_uri(),
        [ 'novai-style' ],
        wp_get_theme()->get( 'Version' )
    );
} );

Frequently Asked Questions

Can I use NovAI without the block editor?

No. NovAI is a Full Site Editing (FSE) block theme. It requires WordPress 6.4+ with the block editor enabled. Classic editor mode is not supported.

Does NovAI work with WooCommerce?

NovAI is WooCommerce-compatible. Full template overrides are planned for v1.1.

Are the fonts GDPR-compliant?

Yes. NovAI self-hosts the Inter font in assets/fonts/inter/. No requests are made to Google Fonts or any external CDN.

How do I update the theme?

For ThemeForest purchases: download the new version and install via Appearance → Themes → Upload Theme. For WordPress.org: use the standard automatic update mechanism.

Can I translate NovAI?

Yes. Use languages/novai.pot as a base. Generate .po and .mo files with Poedit or WP-CLI: wp i18n make-mo languages/
